# CS-300-Projects

The projects in this course focused on designing a course advising program that could efficiently load, store, search, and display course information. To solve this problem, I analyzed different data structures, including vectors, hash tables, and binary search trees, and evaluated their run-time performance and memory usage. Understanding data structures was important because the choice of data structure directly affected how efficiently the program could organize and retrieve information. Ultimately, I selected a binary search tree because it provided a good balance between search efficiency and the ability to print courses in alphanumeric order.

Throughout the course, I encountered challenges with implementing data structures, debugging code, and configuring my development environment. I overcame these roadblocks by testing my code in smaller sections, reviewing course resources, and troubleshooting errors until I identified the cause. This project expanded my approach to software development by encouraging me to think about efficiency and scalability rather than just functionality. It also improved the way I write programs by emphasizing the importance of modular design, meaningful comments, clear naming conventions, and organized code that is maintainable, readable, and adaptable for future modifications.
